The one-sentence idea

Attention is a soft, learned lookup.

Every word asks a question and offers an answer. Attention lets each word gather information from every other word — but weighted by how relevant they are. The clever part: relevance isn't hand-coded. The model learns it, using three projections of each word.

Query — Q

“What am I looking for?”

Your search box. The current word's request for context.

Key — K

“What do I offer?”

Each word's label / index. Matched against the query.

Value — V

“Here's my content.”

The actual information passed along once a match is found.

The librarian analogy

A search engine living inside every word.

Think of looking something up in a library. You hold a Query (“books about space travel”). Every book on the shelf has a Key on its spine. You compare your query against each key — strong matches glow brighter. Then you read the Value: the actual contents, but you spend most of your time on the books that matched best. That weighted blend of contents is exactly what attention returns.

Attention(Q,K,V) = softmax( QKᵀ / √dk ) V
Q·Kᵀ — every query dotted with every key → a grid of raw scores.
÷ √dk — scale down so large vectors don't saturate the softmax.
softmax — turn scores into weights that sum to 1 (the line brightness above).
× V — mix the values by those weights → the new, context-aware vector.
Step by step

What actually happens, in order.

Project every token three ways. Multiply each word embedding by learned matrices WQ, WK, WV to get its query, key and value. x → q, k, v
Score. Dot the current query with every key. A high dot-product means "these two are relevant to each other." score = q · kᵀ
Scale & normalise. Divide by √dk, then softmax so the scores become attention weights between 0 and 1. w = softmax(score / √dₖ)
Blend the values. Take a weighted sum of all value vectors. The output is each word, now enriched with whatever it chose to attend to. out = Σ w · v
Do it many times at once. "Multi-head" attention runs several Q/K/V sets in parallel — one head may track syntax, another long-range references, and so on — then concatenates the results.
