Three ideas stack into the answer. Attention lets every token look at every other token. An LLM is that mechanism stacked deep, pretrained on the internet — fluent but ungrounded. RAG ties L09's retrieval to the LLM so it stops making things up.

I.
The mechanism

Every token looks at every other token.

Click any underlined token. The bars show which other tokens it pays attention to under self-attention. The word "it" resolves to "trophy" or "suitcase" depending on which context wins.

Resolve "it"

In this classic example, "it" can refer to trophy (too big to fit) or suitcase (too small). The trained attention pattern weighs context tokens; the highest-weight key tells you what "it" is. Try clicking "fit" or "big" too.

Query, Key, Value.

Qquery
"What am I looking for?" Each token projects to a query vector. For "it", the query encodes a need to resolve a pronoun.
Kkey
"What do I offer?" Each token also projects to a key vector advertising what it represents. Nouns publish "I am a noun"; verbs, "I am an action".
Vvalue
"What content do I contribute?" Once Q dots K and softmax picks the winners, the weighted sum of V vectors is the attention output.

One matmul plus a softmax. A transformer is just this block, stacked — in all-MiniLM-L6-v2 (6 layers, 22M), BERT-base (12, 110M), GPT-3 (96, 175B). Scale changes capability, not architecture.

II.
A failure mode

Asked cold, the LLM makes things up.

A pretrained LLM has never seen NorthStar's catalogue. Ask about it with no context and it produces confident product names that don't exist — not because it's broken, but because it's doing what it was trained to do: produce plausible text.

✕ Raw prompt · no grounding

The model invents.

user: What summer dresses does NorthStar sell under £80?
"NorthStar offers several great summer dresses under £80, including the Bloom Sundress (£59), the Coastline Maxi (£72), and the Marigold Wrap (£68). All come in floral prints and are popular this season."

Confident, fluent, completely false. None of those products exist in the catalogue. The model has no ground truth to reach for — so it fills the gap with a plausible average of what "a product list" looks like.

✓ RAG · grounded

The model recommends.

user: Catalogue (top-5 by similarity): • Lila Floral Frock £64 • Marina Wrap Sundress £79 • Coral Tie-Strap Gown £88 • Atlas Linen Wrap Dress £52 • Eden Cotton Wrap £45 Recommend a summer dress under £80 using only these.
"Three options under £80 from the catalogue: the Lila Floral Frock (£64) for a daytime look, the Marina Wrap Sundress (£79) for something more polished, or the lighter Atlas Linen Wrap Dress (£52)."

Same model, grounded in retrieved facts. Every product name and price came from the prompt. The LLM brings fluency and reasoning; retrieval brings facts and an audit trail.

III.
The pattern

RAG, in five steps.

L09's semantic search is the engine of step 2. The LLM is just one box in a larger pipeline — that's how you keep it on a leash.

01
→

Embed corpus

Once, offline. Encode every catalogue row with a sentence-transformer. Store 384-dim vectors next to row IDs.

02
⊗

Embed query

At request time. Same model encodes the customer's question into the same 384-dim space.

03
≈

Retrieve top-K

Cosine similarity, take the 5 closest catalogue rows. This is exactly the L09 engine.

04
☉

Augment + generate

Paste the K rows into the prompt. Instruct the LLM to recommend only from them. Run inference.

05
✓

Guard + log

Check the response for product names not in the retrieved set. Log the (query, retrieved, response) triple for evaluation.

IV.
A checklist

Three checks before you put a RAG system in front of a customer.

Skip these and the assistant will demo beautifully and fail in production — the most expensive way to launch a GenAI feature.

1

Is retrieval right?

The LLM can only recommend what retrieval surfaces. Evaluate retrieval on its own first — do the top-K rows contain the products a human would pick?

✕ "Generation looks fluent — ship."✓ "Top-5 contains the right product on 7/8 queries."
2

Is the prompt grounded?

System prompt must say: "recommend only from the products listed below; if nothing fits, say so." Add a safety check that flags any product name not in the retrieved set.

✕ "Be helpful and creative."✓ "Only from the list. Flag invented names."
3

Eval set, not demo?

20–50 (query → acceptable answer) pairs written before tuning. Measure on that set when you change the retriever, K, prompt, or model.

✕ "Demoed my own query — works!"✓ "50-pair eval; measured after each change."

V.
Check yourself

Eight questions from the field notebook.

Click to flip. Numbering matches lesson.md.

▸ Click to revealTwo reasons not to build the shopping assistant on a vanilla RNN.
Long-range forgetting: the RNN's hidden state has limited capacity; by the time it has consumed 30 tokens, the early words may be diluted. Attention lets every output position look directly at every input token — no bottleneck.
No parallelism: RNNs process tokens sequentially. Attention computes all token interactions in parallel via matmul — which is why transformers train on giant data and RNNs effectively don't.
▸ Click to revealQ/K/V intuition — one sentence each.
Query: what the token is looking for. Key: what the token advertises about itself, matched against queries. Value: the content the token contributes once selected — the weighted sum of values is the attention output.
▸ Click to reveal"GenAI is fundamentally different from L03–L08 classifiers." Correct that.
An autoregressive LLM is a classifier whose classes are "the next token". Given input, it outputs a probability over ~50,000 tokens; you pick one (greedily or sampling), append, run again. Same framework as L03's classifier — model produces probabilities, you pick a label — just looped, with a much larger label space.
▸ Click to revealLLM returns confident product names that don't exist. Bug? Fix?
Not a bug — the model is doing what it was trained to do: produce plausible text. No ground truth for the catalogue means it fills the gap with a plausible average. Fix: give it the catalogue at inference time. That is RAG.
▸ Click to reveal"GPT-4 is smart — just paste the whole catalogue." Why RAG anyway?
Cost & latency: 10,000 products per prompt is wasteful. RAG sends only top-5. Quality: LLMs degrade on very long contexts ("lose the needle"). A focused 5-product prompt with grounding produces more reliable answers. RAG separates retrieval (cheap) from generation (LLM).
▸ Click to revealWhy put the catalogue in the user turn, not the system prompt?
A 360M model can drift on long system prompts containing structured data — it tends to treat them as text to continue rather than rules to follow. Framing the catalogue as "here is what we have, answer using only these" in the user turn keeps the grounding constraint and the data adjacent, matching the chat-template patterns the model was instruction-tuned on.
▸ Click to revealDemo works on Sarah's test query. What does she still need?
(a) A held-out eval set of 20–50 realistic (query → acceptable answer) pairs not seen while building. (b) Measurement at both stages: retrieval precision and generation quality. (c) A safety check that flags responses mentioning names not in the retrieved set.
▸ Click to revealShould the L01 sentiment task be "upgraded" to an LLM?
Probably no. L01's pipeline is fast, cheap, deterministic, good enough. An LLM adds latency, cost, non-determinism, and a hallucination surface for zero accuracy gain. Reach for an LLM when the task needs language generation or open-ended reasoning; stick with the specialised model when classification is all you need.
